# AI Platform Engineering 2026

## 定義轉變
2026 年平台工程從 automation 轉向 **agent-based and agentic workflows**。

Futurum Group 原話：
> "2026 will be the year platform engineering shifts from automation to agent-based and agentic workflows."

## 為何重要
- 近半數平台工程團隊已在使用 AI-assisted tools（Futurum 調查）
- 38% 組織正在標準化平台工程方法
- 19% 已進入 mastering 階段
- 這些成熟度讓平台團隊有能力承擔 AI governance 角色

## 核心應用（對 DevSecOps/IaC）

| 應用 | 說明 |
|---|---|
| AI 生成 IaC | LLM 把 high-level spec 轉成 Terraform/K8s/CloudFormation |
| Predictive scaling | 根據歷史 metrics 預測 capacity 需求，自動調整 |
| CI/CD 增強 | 自動生成 tests、self-healing test suites、intelligent release orchestration |
| SRE / Incident response | 自動拉 logs/metrics/traces、預測故障、降低 MTTR |
| Developer experience | LLM copilot + natural language interface + ChatOps bots |
| Policy as Code review | Agent 理解 OPA/Sentinel policy，檢查 PR 合規性 |

## 2026 態勢
- 平台工程從「效率平台」升級為「戰略驅動力」
- Agent 成為可信賴的開發團隊成員
- 成功指標從 productivity 擴展到 AI applications 的部署量
